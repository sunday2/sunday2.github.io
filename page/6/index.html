log4j跟踪(1)

发表于 2017-03-20 | 分类于 java |

日志框架可谓非常重要，虽然严格来说它不是业务逻辑的一部分，但是却是不可缺少的一部分，因为它，我们可以了解程序的运行状态，快速定位错误位置。一直以来，我都觉得人类设计的产品或者术语在生活中都一定能找到原型(唯物主义)，通过生活中的例子我们能更快的接受这个概念.生活中日志的作用就是软件领域中日志框架的作用。程序都是人类设计出来的，那么设计人员就有必要将程序的运行状态告诉别人，至于应该将什么信息（Layout），以什么方式(Appender)告诉别人是设计人员的事情.下面先将log4j的一些基本概念捋清楚.

Appender

log4j中的Appender是一个接口类，这个类是对日志的载体的抽象，就是用什么载体来记录日志。生活中你可一用纸质媒体，可以用视频形式等。log4j实现该接口的类有：

  • org.apache.log4j.ConsoleAppender（控制台）
  • org.apache.log4j.FileAppender（文件）
  • org.apache.log4j.DailyRollingFileAppender（每天产生一个日志文件）
  • org.apache.log4j.RollingFileAppender（文件大小到达指定尺寸的时候产生一个新的文件）
  • org.apache.log4j.WriterAppender（将日志信息以流格式发送到任意指定的地方）

Layout

Layout是对日志内容以及格式的抽象，它定义了要记录什么内容以及什么格式的日志.log4j中Layout是一个抽象类,已经继承该抽象类的有：

  • org.apache.log4j.HTMLLayout（以HTML表格形式布局）
  • org.apache.log4j.PatternLayout（可以灵活地指定布局模式）
  • org.apache.log4j.SimpleLayout（包含日志信息的级别和信息字符串）
  • org.apache.log4j.TTCCLayout（包含日志产生的时间、线程、类别等等信息）

总结一下Logger的配置方式，logger->level->appender->layout.即logger关联 level，level关联了appender，appender关联layout.之间的关系分别是1对多，1对多，1对1.这点是关键,大家一定要记住.
