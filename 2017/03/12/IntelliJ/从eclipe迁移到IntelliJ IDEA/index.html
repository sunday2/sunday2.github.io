从eclipse迁移到IntelliJ IDEA

发表于 2017-03-12 | 分类于 翻译 |

译者注：作为一个java新人，感觉以后IDEA的使用会更加普遍.所以决定好好了解这个IDEA.

概览

从Eclipse切换到IntelliJ IDEA，特别是如果你已经习惯使用Eclipse，那么需要了解两个IDE之间的一些根本区别，包括它们的
用户界面，编译方法，快捷键，项目配置和其他方面。

用户界面

NO workspace

在启动IntelliJ IDEA时，您会注意到的第一件事是它没有工作空间概念。 这意味着您一次只能处理一个项目。 在Eclipse中，通常有一组可能依赖于彼此的项目，在IntelliJ IDEA中，您有一个由一组模块组成的项目。

如果您有多个不相关的项目，可以在单独的窗口中打开它们。
如果您仍然希望在一个窗口中打开几个不相关的项目，作为解决方法，您可以在IntelliJ IDEA中将它们全部配置为模块。

No perspectives

当你切换到IntelliJ IDEA时，第二个大的惊喜是它没有透视图。

这意味着您不需要手动在不同的工作区布局之间切换以执行不同的任务。 IDE遵循您的上下文并自动启动相关工具。

Tool windows

就像在Eclipse中，在IntelliJ IDEA中，你也有工具窗口。 要打开工具窗口，只需在工具窗口栏中单击它：

如果工具窗口栏被隐藏，您可以通过将鼠标悬停在左下角的相应图标上来打开任何工具窗口:

如果要使工具窗口栏暂时可见，可以按Alt（Cmd for OS X）两次并按住它。

如果不想使用鼠标，可以通过相应的快捷键切换到任何工具栏。 要记住的最重要的快捷键是：

  • -Project: Alt+1
  • -Version Control： Alt+9
  • -Terminal： Alt+F12

  • -Window | Store Current Layout as Default
  • -Window | Restore Default Layout (also available via Ctrl+F12)

多个窗口

IntelliJ IDEA中的Windows管理与Eclipse稍有不同。 您无法使用一个项目打开多个窗口，但可以将任意数量的编辑器选项卡分离到单独的窗口中。

启用行号

默认情况下，编辑器中不显示行号。 要启用它们，请转到Settings/Preferences | Editor | General | Appearance | Show line numbers。 在那里你还会发现其他有用的设置。

General workflow（一般工作流程）

No ‘save’ button

是时候公布一些令人真正令人震惊的消息：IntelliJ IDEA没有保存按钮。 由于在IntelliJ IDEA中，您可以撤消重构并从本地历史（local history）记录中还原更改，因此提示您每次是否保存更改是没有意义的。

Compilation（编译）

IntelliJ IDEA编译项目的方式在许多方面都与Eclipse不同。

Auto-compilation(自动编译)

默认情况下，IntelliJ IDEA不会在保存时自动编译项目，因为通常我们不会在IntelliJ IDEA中显式调用保存操作。

如果要模拟Eclipse行为，可以调用Make Project操作（Ctrl + F9） - 它将保存更改的文件并进行编译。 为了方便起见，您甚至可以将Ctrl + S快捷方式重新分配到“Make Project”操作。

要启用自动编译，请导航到 Settings/Preferences | Build, Execution, Deployment | Compiler并选择“ Make project automatically”选项。(译者注：我发现我的版本并没有make projecr这个选项，只有build project automatically)。

请注意，IntelliJ IDEA中的自动编译与Eclipse中的自动编译不同。 在Eclipse中，它不是完全自动的，因为它是由用户显式调用的保存操作触发的，而在IntelliJ IDEA中，当您在编辑器中键入时，它会被隐式调用。

这就是为什么，即使启用了Make project自动选项，IntelliJ IDEA也不会在至少一个应用程序正在运行时执行自动编译：它将隐式地重新加载应用程序中的类。 在这种情况下，您可以调用Build | 制作项目（Ctrl + F9）。
