作业调度神器之QUARTZ初探

发表于 2017-05-29 | 分类于 java |

在一个软件产品中，我们有时不可避免的需要用到定时任务，比如：生成报表，博客定时更新等。所以定时任务的应用是十分广泛的。

目前定时任务的几种实现

根据复杂度的不同，目前定时任务主要有以下几种实现：

  • java.util.Timer
  • java.util.Excutors
  • 开源任务调度框架Quartz
  • 开源任务调度框JCronTab
    开发者可以根据不同需求进行选择。

几种调度的优缺点

Timer（java.util.Timer）

Timer的优点就是简单易用，缺点

1.所有任务都是同一个线程进行调度，所以所有任务都是串行执行的，也就是同一个时刻只有一个任务在执行。

2.只能执行多少时间后调度某个任务，然后每隔多少时长重复执行该任务。更复杂的需要结合Calendar。

Excutors（java.util.Excutors）

因为Timer的缺点，所以java1.5版本增加了Excutors这个基于线程池的调度器，所以Excutors中任务是并发执行的，是基于多线程的。但是同样存在缺点：

1.只能执行多少时间后调度某个任务，然后每隔多少时长重复执行该任务。更复杂的调度需要结合Calendar。

另外，java.util.concurrent.ScheduledThreadPoolExcutor中的启动任务的两个方法：scheduledAtFixedRate和scheduledWithFixedDelay之间的区别是：scheduledAtFixedRate任务执行是从
上一次任务开始起向后推一个时间间隔，ScheduleWithFixedDelay任务执行时刻是从上一次任务结束起向后推一个时间间隔。一个上一次任务开始，一个是上一次任务结束，后者多了一个任务执行时间，所以是不一样的，后者时间间隔受到上一次任务执行时间的影响，是不确定的。

使用Quartz的准备工作

此文的Quartz是基于Quartz2.2.1版本。

至少需要一个名为quartz-x.y.z.jar的核心jar放在classpath下，x.y.z是版本号，另外，这个核心包依赖sl4j-api包，所以classpath下也要增加这个jar。

如果需要将调度数据作持久化处理，那么需要在classpath下增加c3p0的jar包。

如果需要将jobs 建入quartz中，那么需要添加quartz-jobs这个jar。

这些包都可以在quartz的分发包中找到。

如果使用maven添加依赖，那么则如下：

<dependency>

<groupId>org.quartz-scheduler</groupId>
<artifactId>quartz</artifactId>
<version>2.2.1</version>

</dependency>

<dependency>

<groupId>org.quartz-scheduler</groupId>
<artifactId>quartz-jobs</artifactId>
<version>2.2.1</version>

</dependency>

Quartz的几个概念

Scheduler

Scheduler是Quartz的主要接口。Scheduler维持着JobDetail和Trigger的注册表。一旦向Scheduler注册，那么Scheduler就需要负责执行任务，当该任务相关的触发器触发的时候。
Scheduler由SchedulerFactory得到。Quartz中SchedulerFactory同样是接口，已知实现该接口的有DirectSchedulerFactory，StdSchedulerFactory。

Job

Job同样是一个接口，表示要执行的作业或者任务。Quartz中同样也有部分对Job接口的实现，但是一般我们需要自己实现该接口。只要实现excute方法就好。

void execute(JobExecutionContext context)throws JobExecutionException

该方法会被调用当该Job关联的Trigger触发的时候。

Job的实例必须要有一个公有的无参构造函数

(这里有点不明白，按理如果类如果没有显示声明构造函数，那么JVM会为该类自动生成一个无参构造函数，不过当对一个类显示声明了有参构造函数的时候，一般都需要自己添加一个无参构造函数，防止继承的时候需要，这是一个好习惯)。

另外，Quartz是通过JobDetail来装Job和Job的属性。JobDetail可以通过Job的excute方法的参数获得。Quartz没有为Job设计带参数的构造函数，或者说没有为Job(Job是接口)添加属性，因此需要通过额外的JobDataMap 来存储 Job 的属性。JobDataMap 可以存储任意数量的 Key，Value 对。JobDataMap通过JobDetail获得。

Trigger

Trigger同样是一个接口，Job关联的触发器。Quartz同样提供了部分实现了该接口的触发器，经常用到的有CronTriggerImpl,SimpleTriggerImpl。

SimpleTriggerImpl

SimpleTrigger 适用于在某一特定的时间执行一次，或者在某一特定的时间以某一特定时间间隔执行多次。

其中最全面的一个构造函数是：

SimpleTriggerImpl(String name, String group, Date startTime, Date endTime, int repeatCount, long repeatInterval)

当实现某一特定的时间段以某一特定时间间隔执行多次的时候可能repeatCount和endTime会产生冲突，此时优先考虑endTime。

CronTriggerImpl

CronTrigger 的用途更广，相比基于特定时间间隔进行调度安排的 SimpleTrigger，CronTrigger 主要适用于基于日历的调度安排。例如：每星期二的 16:38:10 执行，每月一号执行，以及更复杂的调度安排等。

CronTrigger 同样需要指定 start-time 和 end-time，其核心在于 Cron 表达式，由七个字段组成：

  1. Seconds
  2. Minutes
  3. Hours
  4. Day-of-Month
  5. Month
  6. Day-of-Week
  7. Year(Optional field)

example:

创建一个每三小时执行的 CronTrigger，且从每小时的整点开始执行：

  1. 0 0 0/3 ?

创建一个每十分钟执行的 CronTrigger，且从每小时的第三分钟开始执行：

  1. 0 3/10 * ?

创建一个每周一，周二，周三，周六的晚上 20:00 到 23:00，每半小时执行一次的 CronTrigger：

  1. 0 0/30 20-23 ? * MON-WED,SAT

创建一个每月最后一个周四，中午 11:30-14:30，每小时执行一次的 trigger：

  1. 0 30 11-14/1 ? * 5L

解释一下上述例子中各符号的含义：

首先所有字段都有自己特定的取值，例如，

Seconds 和 Minutes 取值为 0 到 59，

Hours 取值为 0 到 23，

Day-of-Month 取值为 0-31,

Month 取值为 0-11，或者 JAN，FEB, MAR, APR, MAY, JUN, JUL, AUG, SEP, OCT, NOV, DEC，

Days-of-Week 取值为 1-7 或者 SUN, MON, TUE, WED, THU, FRI, SAT。每个字段可以取单个值，多个值，或一个范围，例如 Day-of-Week 可取值为“MON，TUE，SAT”,“MON-FRI”或者“TUE-THU，SUN”。

通配符 表示该字段可接受任何可能取值。例如 Month 字段赋值 表示每个月，Day-of-Week 字段赋值 * 表示一周的每天。

/ 表示开始时刻与间隔时段。例如 Minutes 字段赋值 2/10 表示在一个小时内每 20 分钟执行一次，从第 2 分钟开始。

? 仅适用于 Day-of-Month 和 Day-of-Week。? 表示对该字段不指定特定值。适用于需要对这两个字段中的其中一个指定值，而对另一个不指定值的情况。一般情况下，这两个字段只需对一个赋值。

L 仅适用于 Day-of-Month 和 Day-of-Week。L 用于 Day-of-Month 表示该月最后一天。L 单独用于 Day-of-Week 表示周六，否则表示一个月最后一个星期几，例如 5L 或者 THUL 表示该月最后一个星期四。

W 仅适用于 Day-of-Month，表示离指定日期最近的一个工作日，例如 Day-of-Month 赋值为 10W 表示该月离 10 号最近的一个工作日。

#仅适用于 Day-of-Week，表示该月第 XXX 个星期几。例如 Day-of-Week 赋值为 5#2 或者 THU#2，表示该月第二个星期四。

CronTrigger的使用：

CronTrigger cronTrigger = new CronTrigger("myTrigger", "myGroup"); 
try { 
cronTrigger.setCronExpression("0 0/30 20-13 ? * MON-WED,SAT"); 
}catch (Exception e)
{ 
e.printStackTrace(); 
}

Job 与 Trigger 的松耦合设计是 Quartz 的一大特点，其优点在于同一个 Job 可以绑定多个不同的 Trigger，同一个 Trigger 也可以调度多个 Job，灵活性很强。

一个完整使用Quartz的Demo

  1. 通过SchedulerFactory获得默认的Scheduler
  1. (静态)导入需要的类
  1. 实现Job接口，需要声明一个空的构造函数
  1. 向Scheduler注册Trigger和Job

我们并没有直接得到Job的一个实例，而是通过JobBuilder中的静态newJob方法(这里是静态导入，所以省略类)传递Job实现的class类得到JobDetail实例，并作为scheduler的参数。Trigger同样是通过TriggerBuilder来得到Trigger实例，最后作为Scheduler的参数。
