java基础巩固之File

发表于 2017-04-04 | 分类于 java |

在java的所有原生类库中，File类是比较重要的类之一。

继承体系

只有一个父类，Object。实现了两个接口，Serializable，Comparable。分别说明了File对象可以进行持久化存储，网络传输和两个File对象通过compareTo方法进行比较。

File类简介

File类是对文件和目录路径的抽象表示。

用户界面和操作系统使用系统依赖的路径名命名文件和目录。这个类是对分层路径名的一种抽象的，系统无关的视图表示。

一个路径的抽象由两部分组成。

1.一个可选的系统依赖的前缀字符串，例如磁盘驱动说明符，UNIX系统根目录”/“,微软Windows的UNC路径名”\\“。

2.零个或多个字符串名字的序列。

抽象路径的第一个名字可能是目录名或者，如果是微软Windows的UNC路径名，那么第一个名字是主机名。随后（除了最后一个）的抽象路径名中的名字表示目录名，最后一个名字可能表示目录或文件。总结，名字的含义可以是目录名或者文件名或者主机名。

路径抽象和路径字符串之间的转换是系统依赖的

路径抽象(abstract pathname)–>路径字符串(pathname string):名字之间由默认的分隔符字符分割(seperator character)，而分隔符是由系统属性file.seperator定义的。在java中，由File类的public static修饰的字段seperator,seperatorChar提供。

路径字符串(pathname string)—>路径抽象(abstract pathname),路径抽象中的名字间可能被默认的名字分隔符或者其它底层系统支持的分隔符分割。

绝对路径和相对路径(absolute pathname和relative pathname)

不管是路径的抽象形式还是字符串形式，都分为绝对路径和相对路径。绝对路径是完整的，所以不需要其它信息来定位它所表示的文件。相反，相对路径必须根据从其它路径获取的信息进行解释。通常java.io包里的类是通过用户当前目录(current user directory)来解析相对路径，这个目录通过系统属性user.dir来命名，这个目录一般也是JVM被调用的目录。（你在哪个目录里调用了JVM，执行java程序，那个目录就是用户当前目录）

抽象路径的父亲(parent of abstract pathname)

可以通过File的getParent方法得到抽象路径的父亲，而且这个父亲路径是由抽象路径的前缀和抽象路径中除了最后一个名字的其它名字序列组成。任何目录的绝对路径是任何带有由目录的绝对路径开始的绝对抽象路径的File对象的祖先。例如，由”/usr”表示的抽象路径是由”/usr/local/bin”表示的抽象路径的祖先。

路径前缀

前缀这个概念是用来处理UNIX操作系统中的根目录和微软系统下的驱动说明符，根目录，UNC路径。
如下：

  • UNIX平台中，绝对路径的前缀总是”/“.相对路径没有前缀。表示抽象路径的根目录拥有前缀”/“和空的名字序列。
  • 在微软Windows平台中，路径的前缀由驱动说明符(drive specifier)组成,而驱动说明符是由驱动符和”:”组成即drive letter后面跟着冒号。如果是绝对路径那么驱动说明符后面还跟着”\“。UNC路径的前缀是”\\“,主机名(hostname)和共享名(share name)是跟在后面的名字序列中的最前面的两个。没有指定驱动的相对路径没有前缀。

File类的实例是不可变的; 也就是说，一旦创建，由File对象表示的抽象路径名永远不会改变。

其它经常使用到的，File中的toPath方法可以返回一个java.nio.file.Path对象能够提供更多关于文件对象的操作。RandomAccessFile对象。

为了实现系统无关的文件创建和存储工作，一般我们会使用系统属性，这样就不用考虑路径前缀问题，毕竟路径前缀是系统依赖的：

  • user.home: 用户的home direcrory
  • user.dir: 用户当前工作的目录
  • java.io.tempdir: 默认临时文件目录

这些系统属性是通过System类中的getProperty方法得到。

####File类中的四个public static修饰的字段

  • pathSeperator:是不同抽象路径的分隔符，是String类型。使用情景诸如读取系统环境变量path或者classpath。
  • pathSeperatorChar:同上，是char类型
  • Seperator:是一个抽象路径中的不同名字的分隔符，是String类型
  • seperatorChar：同上，是char类型分
